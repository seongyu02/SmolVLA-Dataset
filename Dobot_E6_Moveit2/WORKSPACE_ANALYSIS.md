# Dobot E6 작업 공간 분석 및 충돌 문제 해결

## 문제 분석

### 1. "max 450mm" 경고의 정체

**450mm는 Dobot E6의 공식 작업 반경(Working Radius)입니다.**

- 공식 스펙: 작업 반경 450mm
- 현재 코드에서 하드코딩된 값: `radius > 450`
- 위치: `dobot_e6_controller.py`, `pick_place_gui.py` 등

**문제점:**
- 단순히 2D 평면의 반경만 체크하고 있음
- 실제 작업 공간은 **원통형이 아니라 복잡한 형태**
- Z 높이와 자세(rx, ry, rz)에 따라 도달 가능 여부가 달라짐

### 2. 충돌 판정이 나는 실제 원인

#### A. 단순 반경 체크의 한계

현재 코드:
```python
radius = np.sqrt(x**2 + y**2)
if radius > 450:
    print("Warning: far from origin")
```

**문제:**
- 2D 평면만 고려 (X, Y만 체크)
- Z 높이를 고려하지 않음
- 자세(rx, ry, rz)를 고려하지 않음
- 실제 작업 공간은 **구형이 아니라 복잡한 형태**

#### B. 실제 작업 공간 형태

Dobot E6의 실제 작업 공간:
- **높은 Z (400-600mm)**: 반경이 작아짐 (약 300-350mm)
- **중간 Z (200-400mm)**: 최대 반경 도달 가능 (약 450mm)
- **낮은 Z (0-200mm)**: 반경이 작아짐 (약 300-400mm)
- **자세에 따라**: 도달 가능 범위가 달라짐

#### C. 내부 파라미터 오류 가능성

1. **카메라-로봇 변환 매트릭스 오류**
   - 변환된 좌표: Y = -350 ~ -450mm (음수)
   - 반경: sqrt(85² + 450²) ≈ 458mm (450mm 초과)
   - 변환 매트릭스가 실제 로봇 좌표계와 맞지 않을 수 있음

2. **로봇 좌표계 기준점 오류**
   - 로봇 베이스 중심이 (0, 0, 0)인지 확인 필요
   - 실제 베이스 위치와 코드의 기준점이 다를 수 있음

3. **Z 높이와 자세 미고려**
   - 같은 (X, Y) 위치라도 Z 높이에 따라 도달 가능 여부가 다름
   - 자세(rx, ry, rz)에 따라 충돌 가능성이 달라짐

## 해결 방안

### 1. 작업 공간 검증 개선

단순 반경 체크 대신:
- Z 높이를 고려한 3D 작업 공간 검증
- 자세(rx, ry, rz)를 고려한 도달 가능 여부 검증
- 현재 위치에서 목표 위치로의 경로 검증

### 2. 동적 작업 공간 계산

Z 높이에 따른 최대 반경:
```python
def get_max_radius_for_z(z_height):
    if z_height < 100:
        return 300  # 낮은 높이: 작은 반경
    elif z_height < 300:
        return 450  # 중간 높이: 최대 반경
    elif z_height < 500:
        return 400  # 높은 높이: 중간 반경
    else:
        return 300  # 매우 높은 높이: 작은 반경
```

### 3. 카메라-로봇 변환 검증

- 변환된 좌표가 실제 로봇 좌표계와 일치하는지 확인
- 로봇 베이스 기준점 확인
- 변환 매트릭스 재캘리브레이션 고려

### 4. IK 솔루션 사전 검증

이동 전에:
- IK 솔루션이 존재하는지 확인
- 관절 한계 내에 있는지 확인
- 자체 충돌 가능성 확인
