# IK 계산 실패 문제 해결 가이드

## 문제 분석

### 현재 상황
- 좌표: X=-15.59mm, Y=-464.48mm, Z=78.39mm
- 반경: 약 464.7mm
- Z 높이: 매우 낮음 (78.39mm)
- 자세: RX=180°, RY=0°, RZ=0°

### IK 실패 원인

1. **낮은 Z + 큰 반경 조합**
   - Z=78.39mm는 매우 낮은 높이
   - 반경 464.7mm는 이 Z 높이에서 도달하기 어려움
   - 로봇의 물리적 제약: 낮은 Z에서는 팔을 펴기 어려움

2. **Y 좌표가 매우 음수 (-464.48mm)**
   - 로봇 뒤쪽으로 너무 멀리 위치
   - 로봇 베이스와의 각도가 너무 큼
   - 관절 한계에 도달할 가능성

3. **고정된 자세 (RX=180°)**
   - 특정 자세에서만 IK 솔루션을 찾으려고 함
   - 다른 자세에서는 도달 가능할 수 있음

4. **현재 위치에서 목표 위치로의 경로 문제**
   - 직접 경로가 충돌하거나 IK 솔루션이 없을 수 있음
   - 중간 waypoint가 필요할 수 있음

## 해결 방안

### 1. Z 높이 증가 (가장 효과적)

**문제**: Z=78.39mm는 너무 낮음
**해결**: Z 높이를 150-250mm로 증가

```python
# 권장 Z 높이
recommended_z = max(target_z, 150)  # 최소 150mm
# 또는
recommended_z = 200  # 안전한 높이
```

**이유**:
- 높은 Z에서는 더 큰 반경 도달 가능
- 로봇 팔이 충돌 없이 움직일 수 있음
- IK 솔루션이 더 쉽게 찾아짐

### 2. 단계적 이동 전략

**전략 1: Z를 먼저 높이기**
1. 현재 위치에서 Z만 높이기 (200-250mm)
2. 목표 XY로 이동
3. Z를 목표 높이로 낮추기

**전략 2: 중간 waypoint 사용**
1. 현재 위치와 목표 위치의 중간점 계산
2. 중간점으로 이동
3. 목표 위치로 이동

### 3. 자세 각도 조정

고정된 RX=180° 대신 다른 각도 시도:
- RX=0° (수직)
- RX=90° (수평)
- RX=180° (기본값)

### 4. 현재 위치 확인

목표 위치로 이동하기 전에:
- 현재 로봇 위치 확인
- 안전한 중간 위치로 먼저 이동
- 그 다음 목표 위치로 이동

## 코드에서 구현된 해결책

### 1. 자동 Waypoint 전략
- IK 실패 시 자동으로 waypoint 생성
- Z를 먼저 높인 후 XY 이동
- 여러 단계로 나누어 이동

### 2. 대안 자세 시도
- 여러 자세 각도 자동 시도
- 가장 적합한 자세 찾기

### 3. 단계적 이동
- 높은 Z로 먼저 이동
- 목표 XY로 이동
- Z를 낮추기

## 사용자 권장 사항

### 즉시 해결 방법

1. **Z 높이 증가**
   - GUI에서 Z 높이를 150-250mm로 조정
   - 낮은 Z는 IK 솔루션을 찾기 어려움

2. **로봇을 안전한 위치로 이동**
   - 홈 위치로 이동
   - 중간 위치로 이동
   - 그 다음 목표 위치로 이동

3. **수동으로 로봇 이동**
   - 로봇을 목표 위치 근처로 수동 이동
   - 그 다음 정밀 위치 조정

### 장기 해결 방법

1. **캘리브레이션 재수행**
   - 더 높은 Z 높이에서 캘리브레이션
   - 작업 공간 중앙 영역에서 캘리브레이션
   - 더 많은 캘리브레이션 점 수집

2. **작업 공간 재확인**
   - 실제 도달 가능한 범위 측정
   - Z 높이별 최대 반경 측정
   - 측정값으로 코드 업데이트

3. **로봇 설정 확인**
   - 로봇 베이스 위치 확인
   - 좌표계 기준점 확인
   - 관절 한계 확인
