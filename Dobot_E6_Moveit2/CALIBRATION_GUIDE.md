# 카메라-로봇 캘리브레이션 가이드

## 문제: 정확한 좌표 추출이 안되는 경우

현재 캘리브레이션이 완벽하지 않아 좌표 추출 정확도가 낮을 수 있습니다. 이 가이드는 정확도를 개선하는 방법을 안내합니다.

## 현재 캘리브레이션 상태 확인

GUI에서 "Check Calibration Accuracy" 버튼을 클릭하여 현재 정확도를 확인할 수 있습니다.

- **평균 오차 < 5mm**: 우수 ✅
- **평균 오차 5-10mm**: 양호 ⚠️
- **평균 오차 10-20mm**: 보통 ⚠️ (재캘리브레이션 권장)
- **평균 오차 > 20mm**: 불량 ❌ (재캘리브레이션 필수)

## 캘리브레이션 정확도 개선 방법

### 1. 더 많은 캘리브레이션 점 추가

**현재**: 4개 점 (최소 요구사항)
**권장**: 6-12개 점

더 많은 점을 사용하면 변환 행렬의 정확도가 향상됩니다.

### 2. 작업 공간 전체에 고르게 분포

**문제**: 현재 캘리브레이션 점들이 비슷한 위치에 몰려있음
- X 범위: 약 20mm
- Y 범위: 약 36mm

**해결**: 작업 공간 전체에 고르게 분포시키기
- 작업 공간의 4개 모서리
- 작업 공간의 중앙
- 작업 공간의 중간 지점들

### 3. 재캘리브레이션 수행

```bash
python camera_robot_calibration.py
```

**절차**:
1. 카메라와 로봇 연결 확인
2. 작업 공간에 마커(체스보드, 원형 마커 등) 배치
3. 로봇을 마커 위치로 이동
4. 카메라에서 마커 클릭하여 픽셀 좌표 기록
5. 로봇 위치를 로봇 좌표로 기록
6. 최소 6개 이상의 점 수집 (권장: 8-12개)
7. 작업 공간 전체에 고르게 분포

### 4. 캘리브레이션 점 배치 전략

**권장 배치**:
```
    [1]--------[2]
     |          |
     |    [5]   |
     |          |
    [3]--------[4]
    
    [6]  [7]  [8]  (중간 지점들)
```

**주의사항**:
- 모든 점이 같은 평면(Z 높이)에 있어야 함
- 점들이 직선상에 있지 않도록 배치
- 작업 공간의 실제 사용 영역을 커버

### 5. 실시간 보정 사용

GUI에서 좌표 변환 시:
- 변환된 좌표를 확인
- 필요시 Z 높이를 수동으로 조정
- "Test Movement" 기능으로 실제 위치 확인 후 보정

## 캘리브레이션 파일 위치

- **변환 매트릭스**: `camera_robot_transform.json`
- **카메라 캘리브레이션**: `hikrobot_calibration_20260126_143821.npz`

## 문제 해결 체크리스트

- [ ] 캘리브레이션 점이 6개 이상인가?
- [ ] 점들이 작업 공간 전체에 고르게 분포되어 있는가?
- [ ] 모든 점이 같은 Z 높이에 있는가?
- [ ] 점들이 직선상에 있지 않은가?
- [ ] 카메라와 로봇이 움직이지 않았는가?
- [ ] 카메라 렌즈가 깨끗한가?
- [ ] 조명 조건이 일정한가?

## 추가 도움말

캘리브레이션 정확도가 여전히 낮다면:
1. 카메라 내부 캘리브레이션 재수행 (`camera_calibration_hikrobot.py`)
2. 더 정밀한 마커 사용 (체스보드 패턴 등)
3. 카메라-로봇 거리 재확인
4. 작업 공간 범위 재확인
