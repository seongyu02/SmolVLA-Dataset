### PyTorch Dataset

```python
import numpy as np
import torch
from torch.utils.data import Dataset
import cv2

class RobotVisionDataset(Dataset):
    def __init__(self, npy_file):
        self.data = np.load(npy_file, allow_pickle=True)
    
    def __len__(self):
        return len(self.data)
    
    def __getitem__(self, idx):
        entry = self.data[idx]
        
        # 이미지 전처리
        image = entry['image']
        image = torch.from_numpy(image).permute(2, 0, 1).float() / 255.0
        
        # 행동 (관절 각도)
        action = torch.from_numpy(entry['joints']).float()
        
        # 그리퍼 상태
        gripper = torch.from_numpy(entry['gripper']).float()
        
        return {
            'image': image,
            'action': action,
            'gripper': gripper
        }
```

## ⚙️ USB 2.0 대역폭 제약

### 문제
- MV-CE050-30UC는 USB 3.0 카메라
- USB 2.0 포트에 연결 시 대역폭 부족 (302.8 Mbps)

### 해결책
프로그램이 자동으로 처리합니다:

```python
# 프레임레이트 10 FPS로 제한
camera.MV_CC_SetFloatValue("AcquisitionFrameRate", 10.0)
```

**결과:**
- ✅ 10 FPS로 안정적 동작
- ✅ 로봇 데이터도 10Hz로 샘플링
- ✅ 완벽한 동기화 유지

### 더 높은 프레임레이트가 필요하다면?
1. USB 3.0 포트에 연결 (파란색 포트)
2. 코드에서 프레임레이트 수정:
   ```python
   camera.MV_CC_SetFloatValue("AcquisitionFrameRate", 30.0)  # 30 FPS
   ```
3. `sleep(0.1)` → `sleep(0.033)` 수정 (로깅 워커)

### 3. 프레임 캡처 오류

**확인:**
- USB 2.0 대역폭 제약?
- 프레임레이트가 너무 높나요?

**해결:**

- 피드백 데이터가 들어오나요?

**해결:**
- `RobotMode` 확인 (5 또는 7이어야 함)
- 로봇 재시작
