## 📁 파일 구성

### 1. `robot_position_logger.py`
로봇을 이동시키면서 실시간 관절값을 기록하는 프로그램

**기능:**
- 로봇 연결 및 활성화
- 4개 포지션 순차 이동
- 실시간 데이터 로깅 (20Hz 샘플링)
- CSV 파일로 자동 저장

### 2. `analyze_robot_log.py`
저장된 로그 파일을 분석하는 도구

**기능:**
- 로그 파일 목록 표시
- 포지션별 통계 분석
- 이동 거리 및 속도 계산
- 시작/종료 위치 비교

### 3. `logs/` 폴더
로그 파일이 저장되는 폴더 (자동 생성)

---

## 🚀 로그 수집

### 1단계: 프로그램 실행

```powershell

```

## 📄 로그 파일 형식

### CSV 파일 구조

| 컬럼명 | 설명 | 단위 | 예시 |
|--------|------|------|------|
| `position_name` | 포지션 이름 | - | Position 1 |
| `timestamp` | 타임스탬프 | - | 2026-01-22 14:30:45.123 |
| `elapsed_time` | 경과 시간 | 초 | 0.050 |
| `j1` ~ `j6` | 관절 각도 | 도(°) | -70.1970 |
| `x`, `y`, `z` | 직교 좌표 | mm | 27.5680 |
| `rx`, `ry`, `rz` | 회전 각도 | 도(°) | -144.1100 |
| `robot_mode` | 로봇 모드 | - | 5 |

### 샘플링 주기
- **20Hz** (초당 20개 데이터 포인트)
- 0.05초마다 1개 데이터 수집

## 🔧 커스터마이징

### 샘플링 주기 변경

`robot_position_logger.py` 파일에서:

```python
# 115번 라인:
sleep(0.05)  # 20Hz 샘플링 (초당 20번)

# 변경 예시:
sleep(0.01)  # 100Hz 샘플링 (초당 100번) - 더 정밀
sleep(0.10)  # 10Hz 샘플링 (초당 10번) - 파일 크기 감소
```

### 로그 항목 추가

`_logging_worker()` 메서드에서 다른 피드백 데이터 추가 가능:
- 디지털 입력 상태
- 디지털 출력 상태
- 전류값
- 토크값
- 온도 등

### 이동 속도 조절

```python
# 352번 라인, 405번 라인:
speed=30,  # 30% → 원하는 속도 (1-100)
accel=30   # 30% → 원하는 가속도 (1-100)
```

## ⚠️ 주의사항

1. **디스크 공간**
   - 장시간 로깅 시 파일 크기가 커질 수 있습니다
   - 1시간 로깅 시 약 50MB 예상

2. **샘플링 주기**
   - 너무 빠른 샘플링은 CPU 부하 증가
   - 권장: 10Hz ~ 50Hz

3. **CSV 파일 열기**
   - Excel에서 열 때 인코딩 UTF-8-BOM 사용
   - 큰 파일은 전문 도구 사용 권장

4. **실시간 분석**
   - 로깅 중에는 분석 도구 사용 불가
   - 로깅 완료 후 분석

## 📈 통계 정보

### 일반적인 로그 크기
- **1개 포지션 (약 3초 이동)**: ~2KB
- **4개 포지션**: ~8KB
- **100회 반복**: ~800KB
- **1시간 연속**: ~50MB

### 데이터 개수
- **20Hz, 3초 이동**: 60개 데이터 포인트
- **20Hz, 1분 이동**: 1,200개 데이터 포인트

## 📞 문제 해결

### 로그 파일이 생성되지 않는 경우
- 로봇 연결 상태 확인
- Enable 성공 여부 확인
- logs 폴더 쓰기 권한 확인

### 데이터가 이상한 경우
- 피드백 포트(30004) 연결 확인
- 샘플링 주기 조정
- 로봇 펌웨어 버전 확인

### 분석 도구가 작동하지 않는 경우
- CSV 파일 형식 확인
- Python 버전 확인 (3.6 이상)
- 파일 경로 확인
