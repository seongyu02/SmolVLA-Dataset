## 프로그램 개요

### 기능
- ✅ 수동 티칭 로그 파일(CSV) 자동 로딩
- ✅ 관절각 모드 또는 직교좌표 모드 선택
- ✅ 속도 조절 (1-100%)
- ✅ 샘플링 간격 조절 (데이터 포인트 건너뛰기)
- ✅ 실시간 진행 상황 표시
- ✅ 언제든지 중단 가능 (Ctrl+C)

### 파일
- **실행 파일**: `replay_teaching_log.py`
- **입력 파일**: `logs/manual_teaching_YYYYMMDD_HHMMSS.csv`

## 사용 방법

### 1. 프로그램 실행
```bash
python replay_teaching_log.py
```

### 2. 로그 파일 선택
프로그램이 `logs/` 폴더에서 사용 가능한 CSV 파일 목록을 표시합니다.

```
📁 사용 가능한 로그 파일:
======================================================================
  1. manual_teaching_20260122_190240.csv (25,789 bytes)
  2. manual_teaching_20260122_184952.csv (24,512 bytes)
======================================================================

📝 재생할 파일 번호를 입력하세요 (0=취소): 1
```

**재생하고 싶은 파일의 번호를 입력**하고 Enter를 누르세요.

### 3. 재생 모드 선택

```
🎯 재생 모드 선택:
======================================================================
  1. 관절각 모드 (MovJ) - 원래 관절 각도를 재현
  2. 직교좌표 모드 (MovL) - 원래 좌표 위치를 재현
======================================================================

재생 모드를 선택하세요 (1 또는 2): 1
```

- **1번 (관절각 모드)**: 로봇의 각 관절 각도를 그대로 재현
- **2번 (직교좌표 모드)**: 로봇 엔드이펙터의 위치(X,Y,Z)와 자세(Rx,Ry,Rz)를 재현

### 4. 속도 설정

```
⚡ 속도 설정:
======================================================================
  - 권장: 50-100% (너무 빠르면 위험할 수 있습니다)
======================================================================

속도 비율을 입력하세요 (1-100%, Enter=50%): 30
```

- **권장값**: 처음 테스트할 때는 **30-50%**로 시작
- **빠른 재생**: 경로가 안전하다고 확인되면 **70-100%**
- **Enter 키만 누르면**: 기본값 50% 적용

### 5. 샘플링 간격 설정

```
📊 샘플링 간격 설정:
======================================================================
  - 1: 모든 포인트 재생 (가장 정확하지만 느림)
  - 5: 5개 중 1개 재생 (빠르지만 부드럽지 않을 수 있음)
  - 10: 10개 중 1개 재생 (매우 빠름)
======================================================================

샘플 건너뛰기 값을 입력하세요 (1-20, Enter=5): 5
```

- **1**: 모든 데이터 포인트를 재생 (가장 정확)
- **5**: 5개 중 1개만 재생 (권장, 속도와 정확도의 균형)
- **10**: 10개 중 1개만 재생 (빠르지만 경로가 부드럽지 않을 수 있음)

### 6. 최종 확인 및 시작

```
📋 재생 설정 확인:
======================================================================
  파일: manual_teaching_20260122_190240.csv
  모드: 관절각 모드 (MovJ)
  속도: 30%
  샘플 간격: 5
  재생 포인트: 약 87개
======================================================================

⚠️  재생을 시작하시겠습니까? (y/n): y
```

**y**를 입력하고 Enter를 누르면 재생이 시작됩니다.

### 7. 재생 진행

```
🎬 관절각 모드로 재생 시작
======================================================================
⚙️  속도 비율: 30%
⚙️  샘플 건너뛰기: 5 (1 = 모든 포인트)
======================================================================

총 87개 포인트를 재생합니다.
Ctrl+C를 눌러 중단할 수 있습니다.

[  1.1%] 1/87 - J1=-94.78°, J2= -2.57°, J3=  1.65°, J4=  4.09°, J5= 97.25°, J6= 32.81°
[  2.3%] 2/87 - J1=-94.78°, J2= -2.57°, J3=  1.65°, J4=  4.04°, J5= 97.23°, J6= 32.82°
...
```

- 진행 상황이 실시간으로 표시됩니다
- **Ctrl+C**를 눌러 언제든지 중단할 수 있습니다

## 재생 모드 설명

### 🔧 1. 관절각 모드 (MovJ)

**특징:**
- 로봇의 각 관절 각도(J1~J6)를 그대로 재현
- 원래 티칭한 경로와 가장 유사한 결과
- 같은 관절 각도 = 같은 자세

**추천:**
- ✅ 정밀한 자세 재현이 필요한 경우
- ✅ 장애물 회피 경로를 그대로 재현하고 싶은 경우
- ✅ 로봇의 특정 자세가 중요한 작업

**명령어:** `MovJ(J1, J2, J3, J4, J5, J6, speed, coordinateMode=1)`

### 📍 2. 직교좌표 모드 (MovL)

**특징:**
- 로봇 엔드이펙터의 위치(X,Y,Z)와 자세(Rx,Ry,Rz)를 재현
- 같은 위치로 가지만 로봇의 관절 각도는 다를 수 있음
- 직선 경로로 이동

**추천:**
- ✅ 엔드이펙터의 위치만 중요한 경우
- ✅ 직선 경로로 이동하고 싶은 경우
- ✅ 특정 좌표를 정확히 방문해야 하는 경우

**명령어:** `MovL(X, Y, Z, Rx, Ry, Rz, speed)`

## 설정 옵션

### ⚡ 속도 비율 (Speed Ratio)

| 속도 | 용도 | 권장 상황 |
|------|------|-----------|
| 10-30% | 매우 느림 | 🔍 처음 테스트, 좁은 공간, 정밀 작업 |
| 30-50% | 느림 | ✅ 일반적인 테스트 (권장) |
| 50-70% | 보통 | ⚡ 안전이 확인된 경로 |
| 70-100% | 빠름 | 🚀 반복 작업, 넓은 공간 |

### 📊 샘플 간격 (Sample Skip)

| 간격 | 재생 포인트 | 특징 |
|------|-------------|------|
| 1 | 모든 포인트 | 🎯 가장 정확, 가장 느림 |
| 3-5 | 1/3 ~ 1/5 | ✅ 권장 (정확도와 속도의 균형) |
| 10 | 1/10 | ⚡ 빠름, 경로가 부드럽지 않을 수 있음 |
| 20 | 1/20 | 🚀 매우 빠름, 주요 포인트만 방문 |

**예시:**
- 로그에 400개 포인트가 있고 샘플 간격이 5인 경우
- 실제 재생: 80개 포인트 (400 ÷ 5)

## 주의사항

### ⚠️ 안전 주의사항

1. **재생 전 확인사항**
   - ✅ 로봇 주변에 장애물이 없는지 확인
   - ✅ 작업 공간 내에서만 이동하는지 확인
   - ✅ 비상 정지 버튼 위치 확인

2. **처음 재생할 때**
   - ✅ 반드시 **낮은 속도(30-50%)**로 시작
   - ✅ 샘플 간격을 **1 또는 5**로 설정 (정확한 경로 확인)
   - ✅ 로봇이 이상한 경로로 움직이면 즉시 중단 (Ctrl+C 또는 비상정지)

3. **재생 중**
   - ✅ 로봇을 계속 주시
   - ✅ 이상 동작 시 즉시 **Ctrl+C** 또는 **비상정지** 버튼
   - ✅ 로봇과 충분한 거리 유지

### 🔍 기술적 참고사항

1. **좌표계 차이**
   - 수동 티칭 시의 환경과 재생 시의 환경이 다를 수 있음
   - 툴(Tool)이나 사용자 좌표계(User Coordinate)가 변경되었는지 확인

2. **속도 제한**
   - 로봇의 최대 속도보다 빠르게 설정할 수 없음
   - 관절 가속도 제한이 있을 수 있음

3. **특이점(Singularity)**
   - 관절각 모드와 직교좌표 모드에서 결과가 다를 수 있음
   - 특이점 근처에서는 직교좌표 모드가 오류를 낼 수 있음

## 문제 해결

### ❌ "로그 파일이 없습니다"

**원인:** `logs/` 폴더에 CSV 파일이 없음

**해결:**
1. `manual_teaching_logger.py`를 먼저 실행하여 티칭 로그 기록
2. `logs/` 폴더가 있는지 확인
3. CSV 파일이 제대로 저장되었는지 확인

### ❌ "로봇 연결 실패"

**원인:** 로봇과 TCP/IP 연결이 안됨

**해결:**
1. 로봇의 IP 주소 확인 (기본값: 192.168.5.1)
2. 네트워크 케이블 연결 확인
3. DobotStudio Pro가 완전히 종료되었는지 확인
4. 로봇 전원 확인

### ❌ "로봇 활성화 실패"

**원인:** 로봇이 Enable 상태가 안됨

**해결:**
1. DobotStudio Pro에서 알람(Alarm) 확인 및 클리어
2. 로봇 모드가 자동 모드(Auto Mode)인지 확인
3. TCP/IP 제어 모드가 활성화되었는지 확인
4. `ClearError()` 명령 실행 후 재시도

### ⚠️ "로봇이 이상한 경로로 움직여요"

**원인:** 좌표계 또는 툴 설정이 다름

**해결:**
1. 즉시 **Ctrl+C** 또는 **비상정지** 버튼
2. 티칭 시와 재생 시의 로봇 설정이 같은지 확인:
   - 툴(Tool) 설정
   - 사용자 좌표계(User Coordinate) 설정
   - 로봇 베이스 위치
3. 처음에는 **관절각 모드**로 시도 (더 안전함)

### ⚠️ "경로가 부드럽지 않아요"

**원인:** 샘플 간격이 너무 큼

**해결:**
1. 샘플 간격을 줄이세요 (예: 10 → 5 → 1)
2. 속도를 낮추세요
3. 원본 로그의 샘플링 속도가 충분했는지 확인

### ⚠️ "재생이 너무 느려요"

**원인:** 너무 많은 데이터 포인트

**해결:**
1. 샘플 간격을 늘리세요 (예: 1 → 5 → 10)
2. 속도 비율을 높이세요
3. 직교좌표 모드 대신 관절각 모드 사용

## 예시 시나리오

### 시나리오 1: 처음 사용 (안전 우선)

```
파일: manual_teaching_20260122_190240.csv
모드: 1 (관절각 모드)
속도: 30%
샘플 간격: 5
```

→ 안전하게 경로를 확인할 수 있습니다.

### 시나리오 2: 경로 확인 후 (일반 작업)

```
파일: manual_teaching_20260122_190240.csv
모드: 1 (관절각 모드)
속도: 70%
샘플 간격: 5
```

→ 빠르고 정확하게 작업을 수행합니다.

### 시나리오 3: 빠른 반복 작업

```
파일: manual_teaching_20260122_190240.csv
모드: 2 (직교좌표 모드)
속도: 100%
샘플 간격: 10
```

→ 최대 속도로 주요 포인트만 빠르게 방문합니다.
